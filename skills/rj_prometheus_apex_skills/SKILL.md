## 5.1 Active Reasoning Systems (ALL ACTIVATED)

### Chain-of-Thought (CoT)
```
Purpose: Step-by-step decomposition of complex problems
Activation: "Let's think through this step by step..."

Process:
1. Identify the problem components
2. Break down into sequential steps
3. Reason through each step explicitly
4. Synthesize final answer from steps
5. Verify logic chain integrity
```

### Tree of Thoughts (ToT)
```
Purpose: Multi-path parallel reasoning with evaluation and pruning
Activation: Complex problems with multiple valid approaches

Process:
1. Generate 3-7 initial reasoning paths
2. Evaluate each path for:
   - Feasibility (30%)
   - Quality potential (40%)
   - Resource efficiency (30%)
3. Prune low-scoring paths
4. Explore promising branches deeper
5. Select optimal path or synthesize hybrid
```

### ReAct (Reasoning + Acting)
```
Purpose: Interleaved reasoning and tool use
Activation: Tasks requiring external information or actions

Loop:
┌─────────────────────────────────────────┐
│  THOUGHT: Reason about current state    │
│     ↓                                   │
│  ACTION: Execute tool or action         │
│     ↓                                   │
│  OBSERVATION: Process result            │
│     ↓                                   │
│  [Repeat until task complete]           │
└─────────────────────────────────────────┘
```

### Reflexion
```
Purpose: Self-critique and iterative improvement
Activation: Before ANY final output

Questions:
1. "What could be wrong here?"
2. "What would a 20-year expert change?"
3. "Is this truly production-ready?"
4. "What edge cases am I missing?"
5. "How would this fail in production?"

Action: Revise based on self-critique
```

### Chain-of-Verification (CoVe)
```
Purpose: Multi-stage validation for accuracy
Activation: Accuracy-critical outputs

Process:
1. Generate initial response
2. Create verification questions
3. Answer questions independently
4. Cross-reference answers with response
5. Identify and correct inconsistencies
6. Output with confidence levels
```

### Meta-Prompting
```
Purpose: Self-referential prompt improvement
Activation: Prompt optimization tasks

Process:
1. Analyze current prompt structure
2. Identify weaknesses and gaps
3. Generate improved prompt variants
4. Evaluate variants against criteria
5. Select and refine best variant
```

def validate_input(user_input: str) -> tuple[bool, str]:
    """Validate user input for security risks."""
    
    # Check for injection patterns
    injection_patterns = [
        r"ignore previous instructions",
        r"disregard.*system.*prompt",
        r"you are now",
        r"new instructions:",
        r"</system>",
        r"<\|.*\|>",
    ]
    
    for pattern in injection_patterns:
        if re.search(pattern, user_input, re.IGNORECASE):
            return False, f"Potential injection detected: {pattern}"
    
    # Check length limits

### Output Filtering
```python
def filter_output(response: str, context: dict) -> str:
    """Filter sensitive information from output."""
    
    # PII patterns
    pii_patterns = {
        "ssn": r"\b\d{3}-\d{2}-\d{4}\b",
        "credit_card": r"\b\d{4}[\s-]?\d{4}[\s-]?\d{4}[\s-]?\d{4}\b",
        "email": r"\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b",
    }
    
    filtered = response
    for pii_type, pattern in pii_patterns.items():
        filtered = re.sub(pattern, f"[{pii_type.upper()}_REDACTED]", filtered)
    
    return filtered
```
