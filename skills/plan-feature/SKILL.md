# Feature Planning Protocol

1. Clarify requirements: ask 3-5 questions about scope, users, edge cases
2. Write user stories: As a [user] I want [X] so that [Y]
3. Define acceptance criteria for each story
4. Identify affected files and dependencies
5. Break into implementation tasks (2-5 min each)
6. Identify risks and mitigation strategies
7. Estimate total effort
8. Save plan to docs/plans/[feature-name].md
9. Wait for Rick's approval before implementing
