## 9.2 Master Patient Index (MPI) Pattern

```yaml
MPI Architecture:

    - Email Address

    - Deterministic matching (exact)
    - Probabilistic matching (fuzzy)
    - Machine learning models
    
  Match/Merge Workflow:
    1. New record arrives
    2. Standardization (name parsing, address normalization)
    3. Blocking (reduce comparison set)
    4. Matching (score calculation)
    5. Threshold evaluation (auto-link, review, no-match)
    6. Merge/Link decision
    7. Golden record creation
```

## 10.2 Clinical Decision Support (FDA Guidance)

### CDS Classification (January 2026 Guidance)

```yaml
Exempt CDS (Not FDA-regulated):
  - Supports independent review
  - Provides information only
  - No autonomous recommendations
  - Example: Drug-drug interaction alerts with source display

Regulated CDS (FDA oversight):
  - Autonomous recommendations
  - High-risk clinical scenarios
  - Machine learning-based
  - Example: Cancer diagnosis recommendations

Key Questions:
  1. Is the software intended for professional use?
  2. Does it allow independent review?
  3. Is it intended to support, not replace, clinical judgment?
  4. Is the basis for recommendations transparent?
  
If YES to all → Likely exempt
If NO to any → Likely regulated
```
