# Performance — Lighthouse 95+ or It Ships Undone

## Lighthouse targets

```
Performance     ≥ 95
Accessibility   = 100
Best Practices  = 100
SEO             = 100
```

## Core Web Vitals

```
LCP  (Largest Contentful Paint)   < 2.5s
INP  (Interaction to Next Paint)  < 200ms
CLS  (Cumulative Layout Shift)    < 0.1
TTFB (Time to First Byte)         < 800ms
FID  (First Input Delay)          < 100ms   (legacy, still tracked)
```

If a build misses these, the build isn't done. No exceptions.

## Image optimization

- `next/image` on every raster image — never a raw `<img>` tag in shipped code
- WebP / AVIF via `next/image`'s automatic format negotiation
- Responsive `srcset` generated by the component (don't hand-roll)
- Blur placeholders on all above-fold images (`placeholder="blur"`)
- `priority` on the LCP image (usually hero), lazy on everything else
- SVGs for icons — never PNG icons in 2026

## Code optimization

- Tree-shake aggressively (Turbopack does this by default)
- Dynamic imports for route-level code splitting: `next/dynamic`
- `Suspense` boundaries around every async subtree
- Client components only where you actually need interactivity — Server Components by default
- Minification + compression on the platform level (Vercel handles this)

## Caching

- Next.js 16 **Cache Components** (`'use cache'` directive) on static / semi-static data
- Static generation for anything that doesn't change per-request
- ISR (`revalidate`) for content that updates on a known cadence
- Browser caching headers on static assets (`Cache-Control: public, max-age=31536000, immutable`)
- CDN edge caching — Vercel / Cloudflare handles this if you don't fight it

## Font loading

- `next/font/google` — subset + self-host automatic, no external `<link>` tags
- `display: 'swap'` on all fonts
- Preload display + sans; lazy-load mono

## Script loading

- `next/script` for third-party JS (analytics, chat widgets)
- Strategy: `afterInteractive` default, `lazyOnload` for anything non-critical
- Never inline `<script>` tags in JSX

## Verify before you call it done

- Run Lighthouse in mobile mode (harder threshold than desktop)
- Run against a production build (`next build && next start`), never dev
- Test on a throttled connection (Fast 3G in DevTools) — the LCP number that matters
